CAPABILITIES

Everything you need to
run your Linux lab

Every operation is a one-shot SSH command — no persistent connection, no residual process left behind on target machines.

▶
Remote Command Execution
Run any shell command on one or many machines simultaneously. Output is streamed live into each machine's terminal tile — stdout and stderr clearly separated.
paramiko · SSH exec_command
🔔
Desktop Notifications
Push custom pop-up notifications to any remote desktop using notify-send. Great for lab-wide announcements, maintenance alerts, or exam reminders.
notify-send over SSH
🔒
Screen Lock
Lock any workstation remotely on demand — ideal for enforcing security policies in labs, classrooms, or shared office environments without physically touching the machine.
loginctl / dm-tool
🔌
Shutdown & Reboot
Trigger graceful shutdown or reboot on any target. Optionally notify users before the action executes so they can save work.
systemctl poweroff/reboot
📂
Remote Application Launch
Open any installed GUI or CLI application on a target machine. Useful for kiosk control, automation workflows, or launching monitoring tools remotely.
DISPLAY env · xvfb
📁
File Transfer — 3 modes
Copy files to all remote machines, copy files back from them, or use sudo copy to write into root-owned directories — all from the src/des panel without typing a command.
Copy to · Copy from · Sudo Copy
👤
User Management
Add or remove user accounts on target machines directly from the toolbar. The optional SUDO and DISP flags let you escalate privileges or control display environment per operation.
useradd / userdel
🖥️
Terminal Tile View
Each connected machine gets its own terminal tile in the grid — showing its IP, live command output, and status. Scales automatically as more machines are added.
PyQt5 / GTK live output

Three ways to copy files

Lab Master's file transfer panel lets you move files between your machine and any remote target using a simple src/des path pair — no terminal command needed.

  • Copy to Remote — push a file or directory from your local machine to all connected target machines at the destination path you specify
  • Copy from Remote — pull a file or directory back from a remote machine to your local destination path
  • Sudo Copy — same as copy to remote but runs the transfer with sudo privileges on the target side, so you can write to root-owned directories
  • Progress tracked live in the status bar — finishing N/N shows completed vs total transfers
  • shfm button opens the full SSH file manager for browsing remote filesystems
WORKFLOW

Three steps. Full control.

Target machines only need sshd running. No agents, no daemons, nothing else to install.

01
Choose your IP list
Load or type a list of target machine IPs. Enter your SSH username and password in the left panel, then click Connect — Lab Master opens a session to every host simultaneously and spawns a live terminal tile for each one.
02
Connect to the machines
Lab Master establishes SSH connections to all IPs in your list at once. Each connected machine appears as its own tile in the grid, showing hostname, IP, and a ready prompt — online machines glow green.
03
Execute & see real-time results
Pick any toolbar action — run a command, send a notification, lock screens, shutdown, transfer files — and watch the output stream live into every tile simultaneously. No waiting, no polling.
📋
IP List
Paste a plain list of IPs or load a saved list file. Lab Master connects to all of them in one click.
⚡
Concurrent execution
Commands run on all machines at once using a thread pool — no sequential waiting, even with dozens of hosts.
🟩
Live terminal tiles
Each machine's tile updates in real time as output arrives — stdout streams in as it happens, not after the command finishes.

GET STARTED

Install Lab Master

Python 3.9+ desktop app. Runs on any Linux desktop with PyQt5 or GTK available.

From Source
pip install
# Clone from GitHub git clone https://github.com/HilbertBlack/lab-master.git cd lab-master # Set up a virtual environment python3 -m venv .venv && source .venv/bin/activate # Install dependencies pip install -r requirements.txt # Launch Lab Master python main.py
# Install from PyPI pip install lab-master # Run lab-master
